# UI 规范差异：抽屉层级

## 当前状态

| 元素 | z-index | 说明 |
|------|---------|------|
| `.review-overlay` | 9000 | 评审浮层根节点 |
| `.highlight-box` / `.drag-rect` | 9100 | 已选元素高亮、框选 |
| `.review-toolbar` | 10000 | 顶部工具栏 |
| `.modal-backdrop` / `.drawer-backdrop` | 10001 | 弹窗/抽屉遮罩 |
| `.review-modal` | 10002 | 评审表单弹窗 |
| `el-drawer` (ElementPlus 默认) | ~2000 | 评审列表、组件树抽屉 |

## 问题

`el-drawer` 默认层级低于 `.highlight-box`/`.drag-rect`，导致抽屉被已选高亮和框选覆盖。

## 变更后

| 元素 | z-index | 说明 |
|------|---------|------|
| `el-drawer`（评审列表/组件树） | 10003 | 显式设置，高于所有评审浮层内部元素 |

## 验收标准

1. 打开「评审列表」抽屉时，已选元素高亮和框选不会覆盖抽屉内容
2. 打开「组件树」抽屉时，已选元素高亮和框选不会覆盖抽屉内容
3. 抽屉遮罩层正常显示，点击遮罩可关闭抽屉
