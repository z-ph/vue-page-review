# UI 规范差异：评审层滚动定位

## 当前状态

- `.review-overlay`：`position: fixed; inset: 0`
- 高亮框/框选框：绝对定位，使用文档坐标（含 `scrollX/scrollY`）
- 无滚动监听

## 问题表现

1. 选中元素后滚动页面，高亮框固定在原视口位置，不再覆盖目标元素。
2. 框选后滚动页面，框选框不随内容移动；滚动后框选时，框选起点/终点与鼠标实际文档位置不一致。

## 变更后

- 元素高亮和框选框仍以文档坐标保存。
- 渲染时按当前滚动位置换算为视口坐标。
- 监听 `scroll` 事件，滚动时刷新所有高亮和框选框位置。
- 元素高亮优先通过保存的 selector 重新查询元素位置；查询不到时回退到文档坐标换算。

## 验收标准

1. 选中任意元素，垂直/水平滚动页面，高亮框始终覆盖该元素。
2. 框选任意区域，滚动页面，框选框始终覆盖原区域对应的内容。
3. 在已滚动的页面中框选新区域，框选预览和最终框选框与鼠标拖动的文档区域一致。
4. 退出评审模式或所有选择清空时，移除滚动监听。
